# TAKARA 전환 계획

## Phase 1. 프로젝트 기반 구축

- Next.js 프로젝트 생성
- 공통 Header/AdminShell/ResultReveal/TicketGrid 컴포넌트 생성
- 페이지 라우팅 구조 확정
- Supabase DB 스키마 초안 작성

## Phase 2. 인증

- Supabase Auth 연결
- 회원가입: 아이디 중복, 닉네임 중복, 이메일 가입
- 로그인/로그아웃
- 관리자 role 검증

## Phase 3. 관리자 쿠지 등록

- 쿠지 기본정보 등록
- 상품 등급/수량 등록
- 번호별 상품 배치
- 쿠지 공개/중지/종료 상태 변경

## Phase 4. 쿠지 구매 흐름

- 대기열 입장
- 1명씩 선택 권한 부여
- 3분 번호 선택 제한
- 서버 RPC로 번호 잠금
- 2분 결제 제한
- 결제 실패/시간초과 시 잠금 해제

## Phase 5. 결과 공개

- 결제 성공 시 order_items 생성
- 하나씩 열기/한번에 열기 상태 서버 저장
- 마지막 상품 공개 후 전체 결과 표시
- 보관함 자동 저장

## Phase 6. 배송/운영

- 보관함 합배송 요청
- 배송지 관리
- 운송장 입력
- 관리자 운영 로그
- 회원 주문/배송 이력
