# TAKARA 구매 흐름 패치

이 패치는 쿠지 등록 이후 고객 구매 흐름을 실제 데이터 기준으로 연결합니다.

## 포함 기능

- 번호 선택 시 서버에서 티켓 예약
- 같은 번호 동시 선택 방지
- 2분 결제 제한 시간
- 결제 완료 시뮬레이션
- 결제 완료 후 티켓 sold 처리
- 주문/결과/보관함 데이터 저장
- 보관함 실제 상품 표시
- 마이페이지 카운트 표시
- 관리자 주문/회원/로그 데이터 표시

## 테스트 순서

1. 일반 사용자 계정으로 로그인
2. `/kuji`에서 진행중 쿠지 접속
3. 번호 선택
4. 결제 페이지에서 결제 완료 시뮬레이션
5. 결과 확인
6. `/storage`에서 당첨 상품 확인
7. 관리자 계정으로 `/admin/orders` 확인

## 주의

현재 결제는 PG 연동 전 단계의 시뮬레이션입니다. 실제 결제 연동 시 `/api/payment/complete`에서 PG 결제 검증 로직을 추가해야 합니다.
